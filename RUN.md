# How to produce / reproduce the evidence

`ping_results.txt` and `verification_flows.json` are generated by a successful
`cluster_setup.sh` run. Pick whichever path matches your machine.

## Option 1 — GitHub Actions (recommended, no local setup)

The repo ships a workflow that runs the whole stack on a KVM-capable Linux runner
and uploads the two evidence files as an artifact. This is how the committed
evidence was produced.

```bash
# from the repo, with the GitHub CLI authenticated
gh workflow run "Capture OVS Evidence"
gh run watch
gh run download --name ovs-evidence -D artifacts
cp artifacts/ovs-evidence/ping_results.txt artifacts/ovs-evidence/verification_flows.json .
```

The `Verify artifacts` step gates the build on two real `0% packet loss` blocks and a
populated `verification_flows.json`, so a green run guarantees genuine evidence.

## Option 2 — Linux host with Docker + `/dev/kvm`

```bash
docker info                       # daemon must be running
ls -la /dev/kvm                   # present => guests boot in minutes
chmod +x cluster_setup.sh
./cluster_setup.sh                # bootstrap + verify + capture
```

Artifacts land next to the script. Tear down with `CLEANUP=1 ./cluster_setup.sh`.

## Option 3 — Apple Silicon / no KVM (slow, best-effort)

`cluster_setup.sh` auto-detects the missing `/dev/kvm`, installs KubeVirt v1.9 with
the `CrossArchitectureVirtualization` gate, and runs the guests as amd64 under QEMU
TCG. Give Docker Desktop ≥ 8 GB RAM. Boot can take tens of minutes; the CI or
KVM path (Options 1–2) is strongly preferred for fast, reliable captures.

## Verify the artifacts

```bash
grep -c "0% packet loss" ping_results.txt        # expect 2
python3 -c "import json; d=json.load(open('verification_flows.json')); assert d['flows'] and d['datapath_flows'] and d['fdb']"
```
